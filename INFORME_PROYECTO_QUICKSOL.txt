1. RESUMEN GENERAL

Quicksol es una plataforma web empresarial para cargar, procesar y consultar
archivos de negocio, especialmente archivos Excel y CSV grandes. El objetivo no
es una demo, sino una base preparada para produccion, con autenticacion real,
roles, base de datos en Supabase, almacenamiento seguro, importaciones en
segundo plano, diagnosticos y documentacion de despliegue.

El sistema esta pensado para que un usuario pueda subir archivos, que esos
archivos se guarden directamente en Supabase Storage y que un worker separado
los procese sin bloquear la aplicacion web principal.

2. COMO ESTA HECHO

Frontend y aplicacion web:
- Next.js con TypeScript.
- Componentes React para carga de archivos, historial, paneles y administracion.
- UI con estados de subida, archivo cargado, en cola, procesando, reintentando,
  cancelado y completado.
- Indicadores de velocidad de subida y ETA para archivos grandes.

Backend y datos:
- Supabase como base principal.
- Supabase Auth para autenticacion.
- Postgres con migraciones SQL.
- Supabase Storage para guardar los archivos cargados.
- RLS y controles del lado servidor para separar acceso admin/empleado.

Importaciones grandes:
- Los archivos no pasan por Render Web.
- La web pide una URL o flujo de subida y el navegador sube directo a Supabase.
- Para archivos mayores o iguales a 100 MB se usa subida resumible.
- El procesamiento se hace en background con un worker separado.
- El worker lee el archivo desde Storage, procesa por lotes y guarda progreso.

Arquitectura de produccion:
- Render Web: sirve Next.js y maneja usuarios/API.
- Render Worker: ejecuta el proceso de importaciones.
- Supabase: Auth, Postgres y Storage.
- Diagnosticos CLI para validar variables, bucket, schema, jobs y heartbeat.

- Se reforzo la subida directa a Supabase Storage.
- Se agrego subida resumible para archivos grandes con tus-js-client.
- Se agregaron variables de entorno:
  - MAX_UPLOAD_SIZE_MB
  - LARGE_UPLOAD_RESUMABLE_THRESHOLD_MB
  - MAX_ROWS_PER_FILE
  - MAX_EXCEL_SHEETS
  - WORKER_STALE_AFTER_MINUTES
  - WORKER_MAX_ATTEMPTS
  - WORKER_HEARTBEAT_INTERVAL_MS
- Se dejo MAX_UPLOAD_SIZE_GB como variable obsoleta/no usada; el sistema avisa
  que debe usarse MAX_UPLOAD_SIZE_MB.

Se fortalecio el worker:
- Claim atomico de jobs para evitar que dos workers tomen el mismo archivo.
- Heartbeat del worker.
- Deteccion y recuperacion de jobs stale.
- Reintentos automaticos.
- Estado retrying.
- Cancelacion de jobs.
- Limpieza de locks al completar, fallar o cancelar.
- Insercion por chunks.
- Division adaptativa de chunks si Supabase responde con statement timeout.

Se agrego una migracion nueva:

Se mejoro la UI:

- Mensajes cuando el archivo queda esperando worker.
- Velocidad de subida y ETA.
- Retry y cancel.

Se agregaron diagnosticos:

Se agrego documentacion:
